# ev Framework

A React framework built for speed and simplicity, leveraging `@tanstack/react-router` for code-based routing and `@tanstack/react-query` for state management.

- **Repository**: [evjs/evjs](https://github.com/evaijs/evjs)
- **Organization**: `@evjs`
- **CLI**: `ev`

## Roadmap

- [x] Stage 1: Client-first SPA
  - [x] `createApp()` factory (Router + Query + DOM)
  - [x] Code-based routing (via TanStack Router)
- [x] Stage 2: React Server Functions (Alpha)
  - [x] Isomorphic RPC calls (`"use server"`)
  - [x] Webpack plugin with zero-config auto-discovery
  - [x] Versioned manifest schema (`manifest.json` v1)
  - [x] Hono-based API server
- [ ] Stage 3: Server-Side Rendering (SSR)
  - [ ] HTML streaming & Hydration
- [ ] Stage 4: React Server Components (RSC)
  - [ ] Server functions returning React DOM streams

## Packages

### `@evjs/runtime`
Core runtime providing isomorphic utilities.
- `client`: `createApp`, `createRoute`, `Outlet`, `Link`, `__ev_rpc`.
- `server`: `createServer`, `createRpcMiddleware`, `registerServerFn`.
- Server powered by [Hono](https://hono.dev) + `@hono/node-server`.

### `@evjs/manifest`
Shared manifest schema types used by webpack-plugin (producer) and runtime (consumer).
- `EvManifest` (versioned, currently v1).
- `ServerFnEntry`, `SsrEntry`, `RscEntry`, `AssetsEntry`.

### `@evjs/cli`
Command-line interface for project management.
- `ev init [name]`: Scaffold a new project from templates.
- `ev dev`: Start development server.
- `ev build`: Build for production.

### `@evjs/webpack-plugin`
Build-time integration for React Server Functions.
- `EvWebpackPlugin`: Zero-config — auto-discovers `"use server"` files on server builds, emits `manifest.json`.
- `server-fn-loader`: Transforms `"use server"` files into RPC stubs (client) or registered handlers (server).

## Key API

### `createApp(options)`
```tsx
import { createApp, createRootRoute } from "@evjs/runtime";
const rootRoute = createRootRoute({ component: Root });
createApp({ routeTree: rootRoute.addChildren([...]) }).render("#app");
```

### `createServer(options?)`
```ts
import { createServer } from "@evjs/runtime/server";
createServer({ port: 3001 }); // Hono-based RPC server
```

### Server Functions (`"use server"`)
```tsx
// api/users.server.ts
"use server";
export async function getUsers() {
  return await db.users.findMany();
}
```
Imported from client as a standard async function. The `@evjs/webpack-plugin` handles the RPC transformation.

## Architecture: Server Functions

- **Client build**: `server-fn-loader` replaces function bodies with `__ev_rpc(fnId, args)` stubs.
- **Server build**: `server-fn-loader` keeps bodies and injects `registerServerFn(fnId, fn)`.
- **EvWebpackPlugin**: On server builds (`target: "node"`), auto-discovers `"use server"` files and injects them as Webpack entries. Emits `manifest.json`.
- **Runtime**: `createServer()` starts a Hono app with RPC middleware at `/api/rpc`.
- **Dev**: Client uses webpack-dev-server (port 3000) with proxy to API server (port 3001).

## Monorepo Workflow

Managed with Turborepo and npm workspaces.
- `npm run build`: Build all libraries and examples.
- `npm run dev`: Concurrent development mode.
- `npm run release:alpha`: Build and publish all packages to `@evjs` scope.
