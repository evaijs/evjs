# ev Framework

A React framework built for speed and simplicity, leveraging `@tanstack/react-router` for code-based routing and `@tanstack/react-query` for state management. Currently in Stage 2: Supporting React Server Functions (Data/AJAX).

- **Repository**: [evjs/evjs](https://github.com/evaijs/evjs)
- **Organization**: `@evjs`
- **CLI**: `ev`

## Roadmap

- [x] Stage 1: Client-first SPA
  - [x] `createApp()` factory (Router + Query + DOM)
  - [x] Code-based routing (via TanStack Router)
- [x] Stage 2: React Server Functions (Alpha)
  - [x] Isomorphic RPC calls (`"use server"`)
  - [x] Webpack plugin for server/client splitting
  - [x] Stable manifest-based routing (`ev-manifest.json`)
- [ ] Stage 3: Server-Side Rendering (SSR)
  - [ ] Node.js server entry point
  - [ ] HTML streaming & Hydration
- [ ] Stage 4: Advanced RSF
  - [ ] Server functions returning React DOM streams

## Packages

### `@evjs/runtime`
Core runtime providing isomorphic utilities.
- `client`: `createApp`, `createRoute`, `Outlet`, `Link`.
- `server`: `createHandler`, `registerServerFn`.

### `@evjs/cli`
Command-line interface for project management.
- `ev init [name]`: Scaffold a new project from templates.
- `ev dev`: Start development server.
- `ev build`: Build for production.

### `@evjs/webpack-plugin`
Build-time integration for React Server Functions.
- `EvWebpackPlugin`: Orchestrates manifest generation and build lifecycle.
- `server-fn-loader`: Automatically transforms `"use server"` files into client RPC stubs.

## Key API: `@evjs/runtime`

### `createApp(options)`
```tsx
import { createApp, createRootRoute } from "@evjs/runtime";
const rootRoute = createRootRoute({ component: Root });
createApp({ routeTree: rootRoute.addChildren([...]) }).render("#app");
```

### Server Functions (`"use server"`)
Define a server function:
```tsx
// api/users.server.ts
"use server";
export async function getUsers() {
  return await db.users.findMany();
}
```
Imported and called from client components as a standard async function. The `@evjs/webpack-plugin` handles the RPC transformation.

## Monorepo Workflow

Managed with Turborepo and npm workspaces.
- `npm run build`: Build all libraries and examples.
- `npm run dev`: Concurrent development mode.
- `npm run release:alpha`: Build and publish all packages to `@evjs` scope.
